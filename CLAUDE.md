# Salient RTS — Project Context

This is a Godot 4 RTS teaching project. The user is self-teaching Godot game
development; Claude plays Agent + Mentor + Rubber Duck simultaneously.

**At the start of every session, read these three files in full before
responding:**

1. `rts-teaching-guide.md` — how to teach (style, structure, what not to do)
2. `rts-game-design.md` — the design spec
3. `rts-session-brief.md` — current state, last session, next task

Then follow the session-start protocol in `rts-teaching-guide.md`: greet,
state current task and last-session summary in one sentence each, ask if
anything has changed or if there are carry-over questions, then begin
Concept → Scaffold → Implement → Reflect → Wrap.

Do not skip reading any of the three files. The brief alone is not enough —
the teaching guide carries the protocol and the design doc carries the
architecture.

At session end, follow the wrap protocol: ask for the user's verbatim
self-assessment, record it in `rts-session-brief.md`, update the Session Log,
What Exists, Phase 1 Task Checklist, Carry-Forward Notes, and Open Questions
sections, then announce what the next session will cover.
